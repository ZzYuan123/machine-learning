# 第一章 机器学习概述

## 什么是机器学习

一句话：

> 让计算机从数据中学习规律，而不是人为编写规则。

机器学习主要分为：

- 监督学习
- 无监督学习
- 半监督学习
- 强化学习

类型	In（输入）	Out（输出）	目的	案例
监督学习(supervised learning)	有标签	有反馈	预测结果	猫狗分类、房价预测
无监督学习(unsupervised learning)	无标签	无反馈	发现潜在结构	“物以类聚，人以群分”
半监督学习(Semi-Supervised Learning)	部分有标签，部分无标签	有反馈	降低数据标记的难度	-
强化学习(reinforcement learning)	决策流程及激励系统	一系列行动	长期利益最大化	学下棋

# 第二章 特征工程

## 特征工程概念

特征工程是: 利用专业背景知识和技巧处理数据 让机器学习算法效果最好 这个过程就是特征工程
ps: 数据和特征决定了机器学习的上限 模型和算法只是逼近这个上限而已

## 特征提取

从原始数据中提取与任务相关的特征 构成特征向量

## 特征预处理

特征对模型产生影响 因量纲问题 有些特征对模型影响很大 有些很小 所以要对特征进行预处理
方法一: 归一化:将数据缩放到 **[0, 1]** 或 **[-1, 1]** 区间。

    公式：![1783513478726](C:\Users\29412\AppData\Roaming\Typora\typora-user-images\1783513478726.png)

    特点 ：有界性：输出范围固定（通常[0,1]）
	                 对异常值敏感：最大值或最小值异常会影响整体
              适合：数据分布有明确边界、需要压缩到固定区间的场景

方法二: 标准化：将数据转化为**均值为0，标准差为1**的标准正态分布。

    公式：![1783513595935](C:\Users\29412\AppData\Roaming\Typora\typora-user-images\1783513595935.png)

    特点 ：无界性：输出没有固定范围
	                 对异常值敏感度相对归一化较小
              适合：数据服从正态分布、有异常值、需要稳定方差的场景

## 特征降维

将原始数据的维度降低 就叫做特征降维
特征降维会使一部分信息丢失 原始数据会发生变化 只保留了最主要的信息

## 特征选择

原始数据特征很多 但是对任务相关是其中一个特征集合子集
特征选择不会改变原来的数据

## 特征组合

通过矩阵运算 把不同的特征矩阵重新组合 拼成一个新的矩阵

# 第三章 线性回归（Linear Regression）

## 线性回归概述

用线性公式来描述多个自变量x或一个自变量x和一个因变量y之间的关系
对这个关系进行建模 从而达到特征预测标签
自变量x相当于特征值 因变量y相当于标签

## 分类

1 一元线性回归: 1个特征列 + 1个标签列
2 多元线性回归: 多个特征列 + 1个标签列

## 模型

一元线性回归
y = wx + b w:权重 b:偏置
多元线性回归
y = w1x1 + w2x2 + w3x3 + ... + wnxn -> w的转置 * x + b

## 损失函数

误差 = 预测值 - 真实值
损失函数(Loss Function) 也叫做成本函数(Cost Function) 也叫做代价函数、目标函数
用于描述每个样本点和其预测值之间的关系 尽可能的让损失函数的值变小
也就是让误差变小 这样线性回归的结果评估越好
问题 如何损失函数最小?
答案:
    思路1 正规方程法 直接求导 一步求导 直接算导数等于0的时候极小值
    思路2 梯度下降法

## 损失函数分类

最小二乘法: loss = (y1'-y1)²+(y2'-y2)²
MSE(Mean Squared Error) 均方误差 loss = ((y1'-y1)²+(y2'-y2)²) / 2
RMSE(Root Mean Squared Error) 均方根误差 loss = sqrt (((y1'-y1)²+(y2'-y2)²) / 2)
MAE(Mean Absolute Error) 平均绝对误差 loss = (abs(y1' - y1) + abs(y2' - y2)) / 2

# 第四章 梯度

梯度就是**函数在某一点，上升最快的方向**，不同场景的理解不一样：

**单变量函数（比如 y=wx+b 的损失函数）**

- 梯度 = 某一点的**导数（切线斜率）**
- 比如：

  - 导数为正：说明往 x 增大的方向走，函数值会变大
  - 导数为负：说明往 x 减小的方向走，函数值会变大
- 👉 所以导数本身，就是让函数**上升最快的方向**

**多变量函数（比如有多个权重 w1,w2... 的损失函数）**

- 梯度 = 所有变量的**偏导数组成的向量**
- ![image-20260521211924025](https://cdn.llfc.club/image-20260521211924025.png)
- 这个向量的方向，就是让损失函数上升最快的方向

## 为什么要加「负号」

这是最关键的一点！

- 梯度本身，是**上升最快的方向**
- 但我们的目标是**让损失函数最小**（也就是找到山谷的最低点）
- 所以：我们要**往梯度的反方向走**，才能下降得最快
- 👉 公式里的 `-` 号，就是这个意思：减去梯度，就是往反方向走

---

## 学习率 α 为什么不能太大也不能太小？

还是用下山比喻：

- ***\*α\** 太小**：你一步迈得特别小，下山走得特别慢，可能要走一万步才到山脚（训练时间长，容易陷入局部最优）
- ***\*α\** 太大**：你一步迈得特别大，直接从山谷的一边跨到另一边，永远在最低点附近来回跳，根本停不下来（损失函数震荡不收敛，甚至越来越大）
- 机器学习里常用 0.001~0.01，就是在 “走得快” 和 “走得稳” 之间找平衡

---

## 总结

1. 随便选一个初始参数（比如 w=0,b=0）
2. 计算当前位置的梯度（看看现在的 “上升最快方向” 是哪）
3. 往梯度的反方向，迈一步（更新参数）
4. 重复步骤 2-3，直到损失函数几乎不再下降（找到最低点）

## 梯度下降法和偏导的关系

![image-20260521215259089](https://cdn.llfc.club/image-20260521215259089.png)

**梯度的理论值需要所有样本参与计算，但实际工程中，我们可以选择用不同数量的样本近似计算梯度**，于是就有了三种分类：

| 算法                                   | 每次更新用多少样本                | 梯度计算方式                                | 优缺点                           |
| -------------------------------------- | --------------------------------- | ------------------------------------------- | -------------------------------- |
| **全梯度下降（FGD）**            | 全部 m 个样本                     | 用上面的公式，对所有样本求和算梯度          | 梯度最准确，但大数据下计算极慢   |
| **随机梯度下降（SGD）**          | 1 个样本                          | 只用 1 个样本的梯度近似整体梯度（去掉求和） | 计算极快，但梯度噪音大，方向不稳 |
| **小批量梯度下降（Mini-batch）** | 一小部分样本（比如 32/64/128 个） | 对这一小批样本求和算梯度                    | 兼顾速度和稳定性，是现在最常用的 |

**为什么不能直接 “求偏导”，非要写求和公式？**

这里有个关键的区别：

- **数学定义上**：梯度是损失函数对参数的偏导数，是一个确定的公式；
- **编程实现上**：这个偏导数的公式，本身就包含了对所有样本的求和操作，必须通过循环 / 求和来实现。

举个例子：

你之前的身高体重数据有 5 个样本，FGD 的梯度计算，就是把这 5 个样本的误差乘特征值，全部加起来再除以 5，得到的才是真正的梯度。

如果用 SGD，就随便挑其中 1 个样本，直接用它的误差乘特征值当梯度。

**总结**

- 你说的 “对损失函数求偏导”，是**梯度的数学本质**；
- PPT 里的求和公式，是**这个偏导数在线性回归里的具体形式**；
- 三种梯度下降，只是在 “用多少样本计算这个梯度” 上做了不同取舍，核心都是用梯度的反方向更新参数。

# 第五章 正则化

### 回顾

   欠拟合: 模型在训练集和测试集表现得效果都不好
   正好拟合: 模型在训练集和测试集表现得都很好
   过拟合: 模型在训练集表现得好，在测试集表现得不好

### 过拟合和欠拟合原因和解决方案:

   产生原因:
      欠拟合: 模型简单
      过拟合: 模型复杂
   解决方案:
      欠拟合: 增加特征，从而增加模型的复杂度
      过拟合: 减少模型的复杂度，手动筛选(减少)特征，L1和L2正则化

### L1和L2正则化介绍:

    目的和思路:
       都是基于惩罚系数来修改特征对应的权重，惩罚系数越大，则修改力度越大，对应的权重越小
    区别:
      L1能将对应w改为0， L2只能将w无限趋近于0
      爬太白山, 带了一个小背包，装了:登山杖，水，汉堡包，衣服，氧气瓶，雨伞，鞋子...，发现背包满了
      L1 正则化:  去掉非必要选项，假设1天爬完，当天去当天回并且天气晴朗->不带雨伞，鞋子，衣服，
      将他们对应的w权重设置为0
      L2 正则化:  换一个非常大的背包，还是这么多物品，但是占用的空间变小了，w权重变小了

    L1和L2核心对比:
                            L1正则化(Lasso回归)                      L2正则化(岭回归)
              惩罚形式:      权重的绝对值之和                           权重的平方之和
              权重结果:    会把不重要的权重设置成0                       会把不重要的权重无限趋近于0
              模型效果:    模型变稀疏了，特征变少了，可解释性变强           模型更平滑，权重均衡，稳定性强
              使用场景:   特征很多，想要自动筛选关键特征，砍不必要特征        特征都有价值，不想砍特征，保留所有特征，避免过拟合

# 第五章 分类指标

## 混淆矩阵

TP

真正例。

FP

假正例。

TN

真负例。

FN

假负例。

## Precision（查准率）

预测正确的正样本比例。

公式：

TP / (TP + FP)

## Recall（召回率）

真实正样本找回来多少。

公式：

TP / (TP + FN)

## F1 Score

Precision 与 Recall 的调和平均。
